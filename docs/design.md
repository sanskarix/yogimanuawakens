# Design System
# Yogi Manu

Version: 1.0

---

# Design Philosophy

This website should not feel like a product.

It should not feel like an influencer website.

It should not feel like a yoga studio.

It should not feel like an online course.

It should feel like entering a peaceful sanctuary.

Every design decision should slow the visitor down.

Silence is a design element.

Whitespace is a design element.

Light is a design element.

Stillness is a design element.

The interface should quietly disappear behind the experience.

---

# Emotional Direction

The website should evoke:

• Peace

• Warmth

• Presence

• Devotion

• Simplicity

• Nature

• Trust

• Humility

Visitors should naturally breathe slower while exploring the site.

Nothing should demand attention.

Everything should gently invite it.

---

# Visual Inspiration

Draw inspiration from:

• Japanese minimalism

• Handmade Indian paper

• Ashrams

• Meditation retreats

• Documentary photography

• Old journals

• Temple stone

• Natural linen

• Warm sunlight

• Quiet forests

Avoid anything that feels futuristic, glossy or corporate.

---

# Overall Aesthetic

The interface should feel handcrafted.

Natural rather than digital.

Warm rather than cold.

Editorial rather than commercial.

Timeless rather than trendy.

---

# Color Philosophy

The palette should never compete with the content.

Color exists to create atmosphere.

Not excitement.

Natural light.

Soft shadows.

No stock imagery.

No dramatic editing.

No HDR.

No floating mandalas.

# Navigation

---

# Motion Principles

Movement should guide attention.

Never entertain.

---

# Animation Style

Everything should ease naturally.

Use long durations.

Nothing should bounce.

Nothing should snap.

---

# Hover Behavior

Very subtle.

Cards lift slightly.

Images scale 1–2%.

Buttons slightly darken.

Links gently underline.

Nothing flashy.

---

# Scroll Behavior

The page should feel cinematic.

Not animated.

The visitor should hardly notice the animations.

---

# Forms

Simple.

Large fields.

Rounded corners.

No heavy borders.

Plenty of spacing.

Friendly error messages.

---

# Accessibility

WCAG AA contrast.

Keyboard navigation.

Reduced motion support.

Visible focus indicators.

Readable font sizes.

Alt text on all imagery.

Semantic HTML.

---

# Mobile Philosophy

The mobile experience is equally important.

No hidden functionality.

No tiny touch targets.

No cramped layouts.

Generous spacing.

Readable typography.

---

# Light Philosophy

Light is treated as a material.

Images should feel illuminated by natural morning or evening light.

Backgrounds should never be pure white.

Shadows should be soft and diffuse.

Avoid harsh contrast.

Allow empty areas to breathe.

The website should feel as though sunlight is quietly entering the room.

---

# Performance Goals

Lighthouse 95+

Fast image loading.

Lazy-loaded media.

Minimal JavaScript.

Use server components whenever possible.

# Cursor Implementation Notes

Do not generate generic SaaS layouts.

Avoid dashboards, glassmorphism, gradients, neon colors or startup-style UI.

Prefer editorial layouts with generous whitespace.

Prefer vertical rhythm over decorative elements.

Animations should always support stillness.

Every page should feel handcrafted rather than generated.

---

# Guiding Principle

When deciding between a more impressive interface and a more peaceful interface–

Always choose the more peaceful one.
